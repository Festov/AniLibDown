# Changelog

## [1.1.0] — 2026-07-20

### Добавлено
- Экран «Продолжить просмотр» в каталоге
- Настройки загрузок: только Wi‑Fi, лимит параллельных загрузок
- Экспорт и импорт привязок Shikimori (JSON)
- Синхронизация номера серии с Shikimori при досмотре
- Picture-in-Picture и AirPlay в плеере
- История поиска и расширенные фильтры каталога (сортировка, год)
- Уведомления о завершении загрузки
- Выбор субтитров в плеере (если доступны в HLS)
- Адаптация под iPad (`NavigationSplitView`) — опционально
- OSLog-диагностика, retry сетевых запросов
- Обновлённый блок Shikimori в карточке тайтла
- Unit-тесты и CI с тестами перед IPA

### Исправлено (после тестирования)
- Фильтр по году: параметры API `f[years][from_year]` / `f[years][to_year]`
- «Продолжить просмотр»: название и постер (без App Groups — Sideloadly)
- Загрузки «только Wi‑Fi»: не создают фантомные элементы в очереди
- Импорт Shikimori с иконкой, как у экспорта
- Убрана EN-локализация (только русский)
- Виджет убран (нужен Xcode + платная подпись)

### Отшлифовано
- Пустые экраны каталога/коллекции/карточки с «Повторить»
- Toast при недоступном видео и отсутствии URL загрузки
- Офлайн-плеер подхватывает любое скачанное качество
- Статусы коллекции не сбрасываются при сетевой ошибке
- Разрешение на уведомления только при первой завершённой загрузке
- Единый User-Agent из версии бандла; версия `x.y.z (build)` в профиле
- Валидация года в фильтрах; a11y для жанров и истории поиска

### Изменено
- Рефакторинг: ViewModel и компоненты карточки релиза в отдельных файлах
- User-Agent берётся из версии бандла

## [1.0.2] — предыдущий релиз

- Интеграция Shikimori (статусы списка)
